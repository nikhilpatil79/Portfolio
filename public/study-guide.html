01

AI / GenAI

the core of your profile

LLMs (Large Language Models)

Neural networks trained on massive text to predict the next token; the engine behind everything else here.

Know

  • Next-token prediction — they don't "know" facts, they predict likely continuations.
  • Tokens — text is split into subword units; billing & context limits are in tokens, not words.
  • Context window — max tokens the model sees at once (prompt + output).
  • Parameters, pre-training vs fine-tuning, temperature, top-p.
  • Hallucination — confident wrong output; mitigated with RAG & grounding.
"What's temperature?"Randomness of sampling. 0 = deterministic/factual, higher = creative/varied.

RAG (Retrieval-Augmented Generation)

Fetch relevant documents from a knowledge base, inject them into the prompt, then let the LLM answer — grounding it in your data.

Pipeline

  • Ingest → parse docs, chunk them, embed each chunk, store vectors.
  • Query → embed question, similarity search for top-k chunks.
  • Generate → stuff retrieved context into prompt, LLM synthesizes answer.
  • Why — reduces hallucination, adds fresh/private knowledge without retraining.
"RAG vs fine-tuning?"RAG = inject knowledge at query time (cheap, updatable). Fine-tuning = bake behavior into weights (costly, for style/format).

Prompt Engineering

Designing inputs to steer LLM output quality, format, and reliability.

Techniques

  • Zero-shot / few-shot — no examples vs. a few examples in the prompt.
  • Chain-of-thought — "think step by step" to improve reasoning.
  • System vs user vs assistant roles.
  • Structured output — force JSON / schema.
  • Guardrails — constraints to reduce hallucination (your resume line).
"How do you reduce hallucination?"Ground with RAG, ask for citations, lower temperature, constrain to "answer only from context, else say you don't know."

Embeddings

Converting text (or images) into fixed-length vectors where semantic similarity = geometric closeness.

Know

  • Dimensionality — e.g. OpenAI text-embedding-3-small = 1536 dims.
  • Cosine similarity — the usual distance metric.
  • Same model must embed both docs & queries.
  • Foundation for semantic search, RAG, clustering, dedup.
"Why cosine not Euclidean?"Cosine measures angle/direction, ignoring magnitude — better for comparing meaning regardless of text length.

Vector Search / Semantic Search

Finding items by meaning rather than keywords, via nearest-neighbor lookup in embedding space.

Know

  • ANN (Approximate Nearest Neighbor) — HNSW, IVF indexes for speed.
  • top-k retrieval + optional re-ranking.
  • Hybrid search — combine keyword (BM25) + vector.
  • Keyword search misses synonyms; semantic doesn't.
"Keyword vs semantic search?"Keyword matches exact strings; semantic matches meaning ("car" ≈ "automobile"). Your resume claims semantic beat keyword on relevance.

Transformers

The neural architecture (2017 "Attention Is All You Need") underlying all modern LLMs.

Know

  • Self-attention — each token weighs its relation to every other token.
  • Encoder (BERT) vs decoder (GPT) vs encoder-decoder (T5).
  • Positional encoding — injects word order.
  • Parallelizable → why they scaled past RNNs/LSTMs.
"Why did transformers beat RNNs?"Attention captures long-range dependencies and trains in parallel; RNNs process sequentially and forget.

LangChain

Framework to compose LLM apps — chains, prompts, memory, retrievers, tools, agents.

Know

  • Chains — sequence LLM calls / steps.
  • Document loaders → splitters → vector stores → retrievers (the RAG toolkit).
  • Memory — persist conversation state.
  • LCEL — LangChain Expression Language for piping components.
"Why LangChain vs raw API?"It standardizes RAG/agent plumbing (loaders, retrievers, memory) so you don't rebuild it per provider.

Multi-Provider LLM Architecture

Your "provider factory" — one interface, swappable model backends (OpenAI, Claude, Gemini, Groq).

Know

  • Factory / adapter pattern — abstract provider differences behind a common contract.
  • Failover & routing — pick model by cost/latency/task.
  • Token usage tracking & cost optimization (your bullet).
  • Normalize streaming, function-calling, and error formats across vendors.
"Why abstract providers?"Avoid vendor lock-in, route cheap tasks to cheap models (Groq/Gemini) and hard ones to premium (Claude/GPT).

The Provider APIs

OpenAI, Claude (Anthropic), Gemini (Google), Groq — the model endpoints you call.

Know each

  • OpenAI — GPT models, function calling, embeddings, broad ecosystem.
  • Claude — strong reasoning/long context, tool use, MCP-native.
  • Gemini — Google, multimodal, large context.
  • Groq — not a model but ultra-fast inference hardware (LPU) serving open models.
"What is Groq?"A hardware inference provider (LPU chips) giving very low latency on open-source models — speed, not a new model family.

Text-to-SQL

LLM translates a natural-language question into a SQL query against your DB.

Know

  • Feed the LLM the schema (tables/columns) in the prompt.
  • Guardrails — read-only, validate/sandbox generated SQL, prevent injection.
  • Optionally execute + summarize results back in language.
  • Few-shot examples of Q→SQL boost accuracy.
"Biggest risk in text-to-SQL?"Unsafe/destructive queries. Mitigate with read-only roles, query validation, and allow-listed tables.

MCP (Model Context Protocol)

Open standard (Anthropic) letting LLMs connect to tools/data via standardized "MCP servers."

Know

  • Client–server — the AI app is the client; each MCP server exposes tools, resources, prompts.
  • Tool isolation — servers sandbox capabilities (your "secure tool isolation" bullet).
  • Standard transport (stdio / HTTP-SSE) so any model reuses any server.
  • Think "USB-C for LLM tools."
"Why MCP over custom tool code?"One protocol = reusable, isolated, swappable tools across any model/host instead of bespoke per-app integrations.

Structured Output & Schema Validation

Forcing the LLM to return machine-parseable data (JSON) matching a defined schema.

Know

  • Function/tool calling & JSON mode.
  • Schema validation — Zod / Pydantic to enforce shape; retry on mismatch.
  • Async batch processing (your bullet) — parallelize many LLM calls.
  • Handles the "LLM returned almost-valid JSON" problem.
"How do you guarantee valid JSON?"Use the provider's structured-output/tool mode + validate with Pydantic/Zod and auto-retry on failure.

Streaming Responses

Sending model output token-by-token as it's generated instead of waiting for the full reply.

Know

  • SSE (Server-Sent Events) or WebSockets to the client.
  • Improves perceived latency / UX.
  • Pair with conversation management — message history, context windows, summarization.
"How is streaming delivered?"Usually SSE — the server flushes chunks; the frontend appends tokens as they arrive.

Agentic AI & Orchestration

your Agentic SDLC project

AI Agents

LLMs that plan, use tools, act, and observe results in a loop to accomplish goals autonomously.

Know

  • ReAct loop — Reason → Act (tool) → Observe → repeat.
  • Tools/function calling — how agents affect the world.
  • Memory (short vs long term) & state.
  • Specialized agents — Orchestrator, CodeGen, Review, QA, Deploy (yours).
"Agent vs a chain?"A chain is fixed steps; an agent decides its own next step dynamically using tools until the goal is met.

Multi-Agent Orchestration

Multiple specialized agents coordinating — delegation, hand-offs, shared context.

Know

  • Orchestrator/supervisor pattern — one agent routes to workers.
  • Sequential vs parallel vs hierarchical topologies.
  • Shared state / message passing between agents.
  • Trade-off: power vs cost, latency, and error compounding.
"Why multiple agents not one big prompt?"Separation of concerns — each agent has focused context/tools, improving reliability and debuggability.

LangGraph

LangChain's library for agent workflows as a graph of nodes/edges with shared state.

Know

  • Nodes = steps, edges = transitions (incl. conditional).
  • State object flows through the graph.
  • Supports cycles — loops, retries, human-in-the-loop.
  • Good for deterministic, controllable agent flows.
"LangGraph vs LangChain?"LangChain chains are linear; LangGraph models stateful, cyclic, branching agent workflows explicitly.

CrewAI

Framework for role-based multi-agent "crews" — agents with roles, goals, and tasks collaborating.

Know

  • Agents (role + goal + backstory), Tasks, Crew, Process (sequential/hierarchical).
  • Higher-level / more opinionated than LangGraph.
  • Good for quickly modeling a "team" of AI workers.
"CrewAI vs LangGraph?"CrewAI = role-based team abstraction (fast to build); LangGraph = low-level graph control (more precise). You used both.

Agentic SDLC Concepts

Automating the software lifecycle with agents — the theme of your flagship project.

Know

  • Agents map to SDLC roles: plan → code → review → test → deploy.
  • RAG over Jira / Confluence / GitHub for project context & consistency.
  • MCP servers for secure tool access (CI, repos).
  • Human-in-the-loop checkpoints for safety.
"How do agents keep architectural consistency?"RAG retrieval from project docs (Pinecone) injects prior decisions so each iteration stays aligned.

Machine Learning & Deep Learning

foundations + your CV bullets

Core ML Concepts

The fundamentals any ML question circles back to.

Know

  • Supervised vs unsupervised vs reinforcement.
  • Train/validation/test split, cross-validation.
  • Overfitting vs underfitting; bias–variance trade-off.
  • Regularization, feature engineering, hyperparameter tuning.
  • Metrics: accuracy, precision, recall, F1, confusion matrix.
"Precision vs recall?"Precision = of predicted positives, how many right. Recall = of actual positives, how many caught. F1 balances both.

Ensemble Models (your sentiment app)

Combining multiple models for better accuracy — you used SVM, Naive Bayes, Random Forest.

Know

  • SVM — finds max-margin decision boundary; kernels for non-linear.
  • Naive Bayes — probabilistic, assumes feature independence; great for text.
  • Random Forest — bagged decision trees; robust, handles non-linearity.
  • Bagging vs boosting vs voting/stacking.
"Why ensemble for sentiment?"Different models capture different patterns; voting reduces individual weaknesses → your 90%+ accuracy claim.

CNNs (Convolutional Neural Nets)

Deep nets for images — your plant-disease detector. Learn spatial features via filters.

Know

  • Convolution + filters/kernels detect edges→shapes→objects.
  • Pooling (downsampling), activation (ReLU), fully-connected head.
  • Transfer learning — fine-tune ResNet/VGG on your dataset.
  • Data augmentation to fight overfitting.
"Why CNN over a plain NN for images?"Convolutions share weights and exploit spatial locality — far fewer params, translation-invariant feature detection.

Text Preprocessing (NLP)

Turning raw text into model-ready features — the front of your sentiment pipeline.

Know

  • Tokenization, stopword removal, stemming/lemmatization.
  • Bag-of-Words, TF-IDF, n-grams.
  • Word embeddings (Word2Vec/GloVe) as an upgrade.
  • Handling class imbalance.
"What's TF-IDF?"Weights a word by frequency in a doc but down-weights words common across all docs — highlights distinctive terms.

TensorFlow & Keras

Google's DL framework; Keras is its high-level API. You used them for CV/DL.

Know

  • Keras — Sequential / functional API, layers, model.fit().
  • Tensors, computational graphs, autodiff.
  • Loss, optimizer (Adam/SGD), epochs, batch size.
  • Callbacks: early stopping, checkpoints.
"TensorFlow vs PyTorch?"PyTorch = dynamic graphs, Pythonic, research-favored. TF = production/deployment ecosystem (TF Serving, Lite). You've touched both.

PyTorch & Scikit-Learn

PyTorch = deep learning; Scikit-Learn = classical ML.

Know

  • PyTorch — tensors, autograd, nn.Module, training loop, dynamic graphs.
  • Scikit-Learn — fit/predict API, pipelines, GridSearchCV, preprocessing, metrics.
  • Scikit for SVM/NB/RandomForest (your ensemble).
"When Scikit-Learn vs a DL framework?"Scikit for classical/tabular/small data; PyTorch/TF for deep nets on images, text, large data.

Hyperparameter Tuning

Systematically searching model settings for best performance — you cite it for 90%+ accuracy.

Know

  • Grid search vs random search vs Bayesian.
  • Params: learning rate, C/gamma (SVM), n_estimators/depth (RF).
  • Tune on validation set, never the test set.
  • GridSearchCV with cross-validation.
"Parameter vs hyperparameter?"Parameters are learned (weights); hyperparameters are set by you before training (learning rate, tree depth).

Vector Databases

Qdrant · Pinecone · pgvector

What a Vector DB Does

Stores embeddings and does fast approximate nearest-neighbor search over them — the memory layer of RAG.

Know

  • Index types — HNSW (graph), IVF, PQ (compression).
  • Distance metrics — cosine, dot, Euclidean.
  • Metadata filtering — combine vector match with WHERE-style filters.
  • Upsert, collections/namespaces, top-k.
"Why not just a normal DB?"Regular DBs can't do efficient similarity search over high-dim vectors; vector DBs use ANN indexes built for it.

Qdrant

Open-source vector DB (Rust) — your Document QA project's store.

Know

  • Collections of points (id + vector + payload).
  • HNSW index; rich payload filtering.
  • Self-hostable or cloud; REST/gRPC APIs.
"Why Qdrant?"Open-source, self-hostable (data privacy), fast HNSW, strong metadata filtering — good for on-prem RAG.

Pinecone

Fully-managed cloud vector DB — used in your Agentic SDLC project.

Know

  • Serverless, managed scaling — no infra to run.
  • Namespaces, metadata filtering, pods/indexes.
  • Trade-off: convenience vs cost & data leaves your infra.
"Qdrant vs Pinecone?"Pinecone = managed/zero-ops but paid & hosted. Qdrant = open-source/self-host = control & privacy. Pick by ops vs control.

PostgreSQL + pgvector

A Postgres extension adding a vector type + similarity operators — vectors inside your SQL DB.

Know

  • One DB for relational + vector data (no separate service).
  • Operators: <-> (L2), <=> (cosine).
  • Indexes: IVFFlat, HNSW.
  • Great when you already run Postgres.
"When pgvector over a dedicated vector DB?"When scale is moderate and you want vectors alongside relational data in one system — simpler stack, transactional.

Chunking

Splitting documents into passages before embedding — quality here makes or breaks RAG.

Know

  • Chunk size & overlap — too big = noisy, too small = lost context.
  • Split by tokens, sentences, or structure (headings).
  • Recursive character splitting (LangChain).
  • Attach metadata (source, page) to each chunk.
"Why does chunking matter?"Retrieval quality depends on it — good chunks keep coherent context so the LLM answers accurately.

Computer Vision

Hazard Detection + plant disease

Object Detection

Locating and classifying objects in an image with bounding boxes — the heart of your hazard system.

Know

  • Detection vs classification vs segmentation.
  • Bounding boxes + confidence + class.
  • IoU (Intersection over Union), NMS (non-max suppression).
  • Metric: mAP (mean Average Precision).
"What's IoU?"Overlap between predicted and true box ÷ their union — measures localization accuracy; threshold decides a "hit."

YOLOv5

"You Only Look Once" — a fast single-stage detector. Your real-time hazard model.

Know

  • Single-stage — one pass = real-time speed.
  • Grid + anchor boxes predict boxes & classes at once.
  • Train on custom labeled data; export for inference.
  • Speed vs accuracy trade-off (n/s/m/l/x sizes).
"Why YOLO for real-time?"Single forward pass over the whole image → high FPS, unlike two-stage detectors (R-CNN) that are accurate but slow.

SSD (Single Shot Detector)

Another single-stage detector you paired with YOLO.

Know

  • Multi-scale feature maps → detects varied object sizes.
  • Default/anchor boxes at each scale.
  • Fast; slightly different speed/accuracy profile vs YOLO.
"YOLO vs SSD?"Both single-stage/real-time. SSD uses multi-scale feature maps; YOLO is often faster. Ensembling reduces false positives.

OpenCV

The classic CV library — frame capture, preprocessing, drawing, real-time video.

Know

  • VideoCapture for live streams; frame-by-frame loop.
  • Resize, color convert, thresholding, contours.
  • Draw boxes/labels on detections.
  • Bridges camera → model → output.
"OpenCV's role in your pipeline?"It grabs frames from the video stream, preprocesses them for the model, and renders detections/alerts back on screen.

Real-Time Video Processing

Running inference on a live stream fast enough to act instantly — your safety/alert use case.

Know

  • FPS / latency budget; frame skipping.
  • Model optimization (smaller model, GPU, batching).
  • False positive reduction via tuning/thresholds.
  • Trigger downstream actions (visual + audio alerts).
"How'd you cut false positives?"Confidence thresholding, domain-specific training data, and temporal consistency (require detection across frames).

Databases: SQL vs NoSQL

MongoDB (document/NoSQL) vs MySQL/PostgreSQL (relational/SQL).

  • Indexing, normalization vs denormalization.
  • Postgres = SQL + JSON + pgvector (versatile).
"When SQL vs NoSQL?"SQL for structured, relational, transactional data; NoSQL for flexible/evolving schemas and scale. Choose by data shape.
